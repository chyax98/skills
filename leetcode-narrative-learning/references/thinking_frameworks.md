# 思维框架库

通用的问题分析和解题思维框架,用于生成"心智模型" ASCII Art。

---

## 1. 通用解题框架

```
┌─────────────────────────────────────────┐
│         理解问题 (Understand)           │
│  • 输入是什么?                          │
│  • 输出是什么?                          │
│  • 约束条件?                            │
│  • 边界情况?                            │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│        寻找模式 (Pattern)               │
│  • 这个问题属于哪种类型?                │
│  • 见过类似的题吗?                      │
│  • 可以用什么数据结构?                  │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│        设计方案 (Design)                │
│  • 暴力解法是什么?                      │
│  • 瓶颈在哪里?                          │
│  • 如何优化?                            │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│        实现验证 (Implement)             │
│  • 编写代码                             │
│  • 测试用例                             │
│  • 边界检查                             │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│        复杂度分析 (Analyze)             │
│  • 时间复杂度                           │
│  • 空间复杂度                           │
│  • 能否进一步优化?                      │
└─────────────────────────────────────────┘
```

## 2. 双指针思维模型

```
    问题空间
    ┌─────────────────┐
    │ [  L ←→ R  ]   │  (对撞指针)
    └─────────────────┘
         ↓
    决策逻辑
    ┌─────────────────┐
    │ 条件 A → 移动 L │
    │ 条件 B → 移动 R │
    │ 其他   → 双移动 │
    └─────────────────┘
         ↓
    收敛至解
    ┌─────────────────┐
    │   L == R  ✓    │
    └─────────────────┘
```

```
    快慢指针
    ┌─────────────────┐
    │ slow  →  fast  │  (fast 走得快)
    └─────────────────┘
         ↓
    应用场景
    ┌─────────────────┐
    │ • 环检测        │
    │ • 寻找中点      │
    │ • 倍速遍历      │
    └─────────────────┘
```

## 3. 滑动窗口思维模型

```
    固定窗口
    ┌───────────────────────┐
    │ [  window  ] →  →  → │
    └───────────────────────┘
    特点: 窗口大小固定

    可变窗口
    ┌───────────────────────┐
    │ [window]  [  win  ]   │
    └───────────────────────┘
         ↓        ↓
      收缩      扩展

    核心步骤:
    1. 扩展 right,加入元素
    2. 判断窗口是否满足条件
    3. 满足 → 记录答案
    4. 不满足/需收缩 → 移动 left
    5. 重复直到遍历完成
```

## 4. 递归思维模型

```
    递归三要素
    ┌─────────────────┐
    │ 1. 递归终止条件 │  (Base Case)
    │ 2. 递归调用     │  (Recursive Call)
    │ 3. 结果合并     │  (Combine)
    └─────────────────┘

    示例: 树的递归
    ┌─────────────────┐
    │      root       │
    │      /  \       │
    │   left  right   │
    └─────────────────┘
         ↓
    f(root) = 处理(root) + f(left) + f(right)
```

## 5. 动态规划思维模型

```
    DP 四步曲
    ┌─────────────────┐
    │ 1. 定义状态     │  dp[i] 表示什么?
    └─────────────────┘
         ↓
    ┌─────────────────┐
    │ 2. 状态转移方程 │  dp[i] = f(dp[i-1], ...)
    └─────────────────┘
         ↓
    ┌─────────────────┐
    │ 3. 初始化       │  dp[0] = ?
    └─────────────────┘
         ↓
    ┌─────────────────┐
    │ 4. 遍历顺序     │  从小到大 / 从大到小
    └─────────────────┘

    记忆化搜索 vs 递推
    ┌──────────────┐        ┌──────────────┐
    │ 递归 + memo  │        │ 迭代 + dp[]  │
    │ (Top-Down)   │   VS   │ (Bottom-Up)  │
    └──────────────┘        └──────────────┘
```

## 7. BFS/DFS 思维模型

```
    BFS (广度优先)
    ┌─────────────────┐
    │     Queue       │
    │   ┌───┬───┬───┐ │
    │   │ 1 │ 2 │ 3 │ │  (层序遍历)
    │   └───┴───┴───┘ │
    └─────────────────┘

    应用: 最短路径, 层次遍历

    DFS (深度优先)
    ┌─────────────────┐
    │     Stack       │
    │   ┌───┐         │
    │   │ 1 │         │  (深入到底)
    │   │ 2 │         │
    │   │ 3 │         │
    │   └───┘         │
    └─────────────────┘

    应用: 路径查找, 连通性
```

## 8. 二分查找思维模型

```
    查找空间
    ┌───────────────────┐
    │ [L      M      R] │
    └───────────────────┘
         ↓
    判断 target vs M
    ┌───────────────────┐
    │ target < M → 左半 │
    │ target > M → 右半 │
    │ target = M → 找到 │
    └───────────────────┘
         ↓
    收缩边界
    ┌───────────────────┐
    │ L = M + 1  或     │
    │ R = M - 1         │
    └───────────────────┘

    变体: 查找边界
    • 左边界: 最左边的满足条件的位置
    • 右边界: 最右边的满足条件的位置
```

## 9. 哈希表思维模型

```
    空间换时间
    ┌─────────────────┐
    │  Array / List   │  O(n) 查找
    └─────────────────┘
         ↓ 转换
    ┌─────────────────┐
    │   Hash Map      │  O(1) 查找
    │ key → value     │
    └─────────────────┘

    典型应用:
    • 存在性检查: set
    • 计数: map<key, count>
    • 映射关系: map<key, value>
    • 去重: set
```

## 10. 贪心思维模型

```
    局部最优 → 全局最优
    ┌─────────────────┐
    │  每步选择       │
    │  局部最优解     │
    └─────────────────┘
         ↓
    ┌─────────────────┐
    │  证明:          │
    │  局部最优       │
    │  = 全局最优     │
    └─────────────────┘

    关键: 证明贪心策略的正确性
```

## 11. 单调栈思维模型

```
    栈的单调性
    ┌─────────────────┐
    │    栈顶         │  最小/最大
    │     ↑           │
    │     |           │
    │     |           │
    │    栈底         │
    └─────────────────┘

    应用场景:
    • 下一个更大元素
    • 柱状图最大矩形
    • 接雨水问题

    模板:
    for 元素 in 数组:
        while 栈非空 and 栈顶 vs 元素:
            弹出栈顶
            处理
        入栈
```

    ┌───┬───┬───┬───┬───┐
    │ 1 │ 2 │ 3 │ 4 │ 5 │
    └───┴───┴───┴───┴───┘

    前缀和数组

    │ 1 │ 3 │ 6 │10 │15 │
    └───┴───┴───┴───┴───┘

    区间和 [i, j] = prefix[j] - prefix[i-1]

    应用: O(1) 查询区间和
```

## 13. 空间换时间思维模型

```
    朴素方法
    ┌─────────────────┐
    │ 时间: O(n²)     │
    │ 空间: O(1)      │
    └─────────────────┘
         ↓ 优化
    ┌─────────────────┐
    │ 时间: O(n)      │
    │ 空间: O(n)      │  (哈希表/DP数组)
    └─────────────────┘

    权衡: 空间复杂度 vs 时间复杂度
```

## 14. 时间换空间思维模型

```
    记忆化方法
    ┌─────────────────┐
    │ 时间: O(n)      │
    │ 空间: O(n)      │
    └─────────────────┘
         ↓ 优化

## 使用指南

在生成"心智模型"页面时:

1. **识别题目的核心思维模式**
2. **选择最相关的框架 1-2 个**
3. **根据具体题目调整细节**
4. **确保 ASCII Art 简洁清晰**
5. **配合文字说明,解释框架如何应用于当前题目**

示例:
```
题目: Two Sum

选择框架: 哈希表思维模型 + 空间换时间模型

生成 ASCII Art:
    暴力法: O(n²) 查找
    ┌─────────────────┐
    │ 双重循环        │
    │ 时间: O(n²)     │
    │ 空间: O(1)      │
    └─────────────────┘
         ↓ 优化
    哈希表: O(n) 查找
    ┌─────────────────┐
    │ num → index     │
    │ 时间: O(n)      │
    │ 空间: O(n)      │
    └─────────────────┘

配合说明:
"这是经典的空间换时间策略。我们用哈希表记录每个数字的位置,
将查找从 O(n) 降低到 O(1),总体时间复杂度从 O(n²) 优化到 O(n)。"
```
